根据 CI 失败分析结果，在 `issues/` 目录下创建标准格式的 issue 记录文件。

## 前置条件

需要已知以下信息（通常来自 `/analyze-failure` 的输出）：
- 失败日期
- 受影响文件和 API
- 错误摘要和根本原因

如果上述信息不明确，先运行 `/analyze-failure` 获取。

## 执行步骤

1. 确定今日日期（`date +%Y-%m-%d`）
2. 查看 `issues/` 目录下已有文件，确定当天的编号（如当天已有 001 则用 002，以此类推）
3. 按命名规则创建文件：`issues/YYYY-MM-DD-NNN-<简短描述>.md`
4. 文件内容使用以下模板：

```markdown
# [YYYY-MM-DD-NNN] <问题标题>

- **发现日期**：YYYY-MM-DD
- **编号**：YYYY-MM-DD-NNN
- **严重级别**：🔴 编译失败 / 🟡 警告 / 🟢 运行时问题
- **受影响文件**：`相对路径`
- **触发版本**：PyTorch nightly YYYY-MM-DD
- **对应 patch**：`patches/NNNN-<描述>.patch`（如有）

---

## 问题描述

（一段话说明现象）

---

## 根本原因分析

### 1. <原因标题>

（详细说明）

```错误日志关键行```

---

## 修复方案

（说明 patch 的核心改动，或待修复）
```

5. 创建完成后输出文件路径，提示用户可运行 `/gen-patch` 生成对应修复
